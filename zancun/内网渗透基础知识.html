内网渗透相关基础知识

本章介绍一些与内网渗透相关的基础知识，如hash传递，黄金票据等等我们在做与内网渗透相关工作经常会遇到的

黄金票据/白银票据

首先讲一下什么是KDC（Key Distribution Center）密钥分发中心，在KDC中分为两部分，一部分为AS(Authentication Service) 身份验证服务，另一部分为TGS(Ticket Granting Service)票据授权服务。

首先是AS服务，client会先表明自己的请求需求而且会使用自己提供的密码进行加密，AS会用client提供的密码来对请求进行解密，之后就会给client用户发送一个Ticket Granting Ticket票据授予票据,TGT 使用 KDC 的密钥进行加密,可以储存在client本地内存中，然后等client用户拿到TGT之后就可以使用TGT去访问TGS。

TGT 的主要作用是允许用户在不需要每次都输入密码的情况下，从 KDC 获取访问其他服务的票据（Service Tickets）。 换句话说，TGT 就像一个“通行证”，用于证明用户已经通过了初始身份验证，可以请求访问特定服务的权限。

最后是TGS服务，接AS服务，用户得到TGT之后使用TGT去访问TGS，TGS验证TGT后（使用密钥解密），返回一个ST(Service Ticket）)给用户，client拿这个ST去访问server服务端，Server收到Ticket并且KDC进行验证通过后就可以提供服务

简化一下过程可以分为三部分:

Client 与 AS 的交互

Client 与 TGS 的交互

Client 与 Server 的交互

上面提到的TGT就是黄金票据，因为 TGT 可以获得服务资源，因此可以伪造TGT来实现任何资源服务的访问，无需提供有效的账号和密码，最终导致攻击者可以完全控制整个域，因为他们可以冒充任何用户并访问任何资源。

而上面提到的ST就是白银票据，ST是特定服务的票据，通过伪造ST可以实现特定资源服务的访问。

白银票据也就对应着域普通用户的票据，而黄金票据对应着域管理员的票据。

在net user查看域内用户时，几乎都会看到有一个用户叫krbtgt，他就是KDC秘钥分发中心用的超管账户，在创建域时都是会自动创建的。

mimikatz可以抓取krbtgt的哈希值制作成黄金票据

hash传递

PTH Pass The Hash

mimikatz可以实现Hash读取，在域环境中，我们是可以用域管理员的账密hash值来伪造管理员登陆的，这也就是hash传递(PTH),即NTLM密文传递到验证登录的机器，绕过正常验证进行登录系统。

渗透测试过程中，就算内网的机器打了KB2871997补丁，同时注册表中关闭Wdigest Auth，也可以抓取到hash值。

注册表中的Wdigest功能就关系着内存中是否有明文密码，键值为1就是开启

IPC$

IPC$是Windows 系统中一个重要的管理共享，用于在计算机之间或同一计算机上的不同进程之间进行通信，支持进程间通信和远程管理。 虽然它对于 Windows 操作系统的正常运行至关重要，但也可能成为攻击者的目标。

net share 即可看到自己计算机中的IPC$共享,IPC$通常不会显示在资源管理器中，因为它是一个隐藏的共享

IPC$的使用条件:

1.Windows 防火墙允许 IPC$ 共享所需的端口（例如，139 和 445）上的通信

2.服务器上必须运行 Server 服务才能共享资源，包括 IPC$ 共享

3.客户端计算机必须提供有效的用户名和密码才能连接到服务器的 IPC$ 共享

在内网中，默认就会开启IPC$共享文件服务，一般默认共享的是c盘
